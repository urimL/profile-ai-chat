# Project 1. FLOFI

## 프로젝트명
FLOFI

## 한 줄 설명
Style Transfer 기술을 활용한 음악 변환 및 스트리밍 플랫폼 

## 프로젝트 개요
FLOFI는 사용자가 업로드한 MIDI 파일을 딥러닝 모델인 Groove2Groove를 통해 Lo-Fi 스타일 음악으로 변환해주는 서비스입니다.  
단순히 변환 결과를 제공하는 데 그치지 않고, 사용자가 변환된 음원을 자신의 취향에 맞게 실시간으로 커스터마이징할 수 있도록 구성했습니다.  
또한 집중과 몰입을 돕기 위해 뽀모도로 타이머, 백색 소음 등의 편의 기능도 함께 제공했습니다.  
이 프로젝트는 캡스톤 디자인 및 창업 프로젝트에서 최우수상을 수상했습니다. 

## 주요 기능
- MIDI 파일 업로드
- Groove2Groove 기반 Lo-Fi 스타일 변환
- BPM, 리버브 등 실시간 오디오 커스터마이징
- 뽀모도로 타이머 기능
- TODO 리스트 기능
- 백색 소음 기능 

## 기술 스택
- Flask
- Groove2Groove (AI)
- JavaScript
- Tone.js 

## 기술 선택 이유
### Flask
AI 모델과의 연동을 위해 Python 기반 프레임워크가 적합하다고 판단했습니다.  
경량 프레임워크인 Flask를 사용해 서버와 모델 간 데이터 흐름을 단순하게 구성하고, 빠르게 API 형태로 연동할 수 있도록 했습니다. 

### Groove2Groove
기존 MIDI 파일을 단순 재생하는 것이 아니라, Lo-Fi 특유의 감성을 입힌 결과물을 생성하기 위해 스타일 변환 모델을 활용했습니다.  
이를 통해 기존 음악 파일에 새로운 가치를 부여할 수 있었고, 서비스 차별화 포인트를 만들 수 있었습니다. 

### JavaScript / Tone.js
브라우저 환경에서 MIDI 데이터를 분석하고 오디오 효과를 실시간으로 제어하기 위해 Tone.js를 활용했습니다.  
이를 통해 사용자가 직접 BPM이나 이펙트를 조절하면서 즉각적인 결과를 확인할 수 있는 인터랙션을 구현했습니다. 

## 담당 역할
- 팀 리더 역할 수행
- Github, Notion, 간트차트를 활용한 협업 프로세스 리딩
- 프론트엔드 개발
- MIDI 데이터 분석 및 조작 로직 구현
- 사용자 맞춤 오디오 이펙트 시스템 구축
- Groove2Groove AI 모델 API 연동
- 뽀모도로 타이머, TODO 리스트, 백색소음 등 편의 기능 구현 

## 문제 해결 경험
### 주제
정밀한 오디오 컨트롤을 위한 데이터 포맷 선정

### 문제 상황
초기에는 일반적인 오디오 파일 포맷을 활용하는 방안도 고려할 수 있었지만, WAV와 같은 오디오 파일은 이미 고정된 음원 데이터이기 때문에 BPM이나 노트 단위의 세밀한 조작에 한계가 있었습니다.  
즉, 사용자가 직접 음악의 흐름을 세부적으로 커스터마이징하는 기능을 구현하기에 적합하지 않았습니다. 

### 해결 방식
오디오 신호 자체가 아니라, 음악 정보(노트, 길이, 템포 등)를 담고 있는 MIDI 형식을 채택했습니다.  
그리고 이 MIDI 데이터를 JSON 형태로 변환 및 분석하여, 실시간으로 BPM 조절이나 커스텀 효과를 줄 수 있는 구조를 만들었습니다. 

### 의미
이 경험은 단순히 기능을 구현하는 수준을 넘어, 요구사항에 맞는 데이터 포맷과 구조를 선택하는 것이 얼마나 중요한지 배운 사례입니다.  
기술을 사용할 수 있는지보다 서비스 목표에 적합한지 판단하는 역량을 보여주는 프로젝트입니다.

# Project 2. Movie Web

## 프로젝트명
Movie Web

## 한 줄 설명
영화 산업 빅데이터 분석 및 시각화 플랫폼 

## 프로젝트 개요
Movie Web은 영화 산업 관련 통계 데이터를 연도별, 분기별, 국가별로 시각화하여 보여주는 플랫폼입니다.  
또한 감독, 장르, 관객 수 등 다양한 조건으로 데이터를 필터링할 수 있으며, 사용자 맞춤형 마이페이지 기능도 제공했습니다.  
단순 조회형 서비스가 아니라, 영화 산업 데이터를 보다 구조적으로 탐색하고 이해할 수 있도록 돕는 데 초점을 두었습니다. 

## 주요 기능
- 연도별 영화 산업 데이터 시각화
- 분기별 / 국가별 통계 조회
- 감독별 / 장르별 / 관객 수 기반 필터링
- 선호 장르 저장
- 영화 좋아요 저장
- 사용자 맞춤 마이페이지 기능 

## 기술 스택
- PHP (SSR)
- MariaDB 

## 기술 선택 이유
### PHP (SSR)
서버 사이드 렌더링 구조를 통해 초기 렌더링 속도를 개선하고, 데이터 기반 페이지를 빠르게 구성할 수 있도록 했습니다.  
또한 검색 엔진 최적화 측면에서도 SSR 구조가 장점이 있다고 판단했습니다. 

### MariaDB
집계 함수와 복합 조인이 포함된 쿼리를 효율적으로 처리할 수 있도록 MariaDB를 사용했습니다.  
통계 데이터 조회와 필터링이 핵심인 서비스 특성상, 관계형 데이터베이스의 구조적 장점을 살릴 수 있었습니다. 

## 담당 역할
- DB 스키마 설계
- ERD 설계
- 외래키 제약조건 설정을 통한 데이터 무결성 확보
- Roll-up, window 함수 기반 통계 쿼리 구현
- 사용자 로그인 및 개인화 데이터 관리 기능 개발
- PHP 세션 기반 인증 처리
- SSR 기반 통계 시각화 페이지 개발 

## 문제 해결 경험
### 주제
트랜잭션 도입을 통한 데이터 일관성 보장

### 문제 상황
선호 장르 수정과 같은 기능은 여러 테이블 혹은 여러 쿼리가 함께 반영되어야 하는 경우가 있었습니다.  
이때 일부 쿼리만 반영되고 중간에 실패하면, 데이터 불일치가 발생할 가능성이 있었습니다. 

### 해결 방식
`mysqli_begin_transaction`을 적용하여 관련 쿼리들이 모두 성공했을 때만 Commit되도록 설계했습니다.  
이를 통해 부분 반영 문제를 방지하고, 데이터 일관성을 확보했습니다.

### 의미
이 경험은 DB를 단순 저장소로 사용하는 것이 아니라, 데이터 무결성과 일관성을 보장해야 하는 시스템의 핵심 요소로 다룬 사례입니다.  
특히 여러 데이터 변경이 함께 일어나는 상황에서 트랜잭션의 필요성을 직접 체감했습니다.

# Project 3. Workfolio

## 프로젝트명
Workfolio

## 한 줄 설명
취업 준비생들을 위한 통합 플랫폼 

## 프로젝트 개요
Workfolio는 취업 준비생들이 기업 분석 자료, 자기소개서, 자격증, 일정 등을 한 곳에서 체계적으로 관리할 수 있도록 만든 통합 플랫폼입니다.  
특정 기업에 대한 예상 질문과 답변을 정리하고 연습할 수 있으며, Google Calendar API를 연동해 일정을 실시간으로 관리할 수 있도록 구성했습니다.  
취업 준비 과정에서 여러 서비스에 흩어져 있던 정보를 통합해 사용자 편의성을 높이는 데 집중한 프로젝트입니다. 

## 주요 기능
- 기업 분석 데이터 관리
- 자기소개서 관리
- 자격증 및 스펙 관리
- 예상 질문/답변 정리 및 연습
- 일정 관리
- Google Calendar API 연동 

## 기술 스택
- Spring Boot
- RESTful API
- JWT
- Spring Security
- OAuth2
- MySQL
- Redis 

## 기술 선택 이유
### Spring Boot
Auto Configuration을 통해 개발 환경을 빠르게 구축하고, 비즈니스 로직 설계에 더 집중할 수 있었습니다.  
또한 내장 Tomcat을 통해 별도 외부 WAS 설정 없이 독립적인 실행 환경을 구성할 수 있어 개발 및 배포 편의성이 높았습니다. 

### RESTful API
자기소개서, 일정, 기업 분석 등 다양한 자원을 URI와 HTTP 메서드로 명확하게 구분해 설계했습니다.  
프론트엔드와 역할을 분리하고, 확장 가능하고 일관된 API 구조를 만들기 위해 RESTful 방식을 채택했습니다. 

### JWT
공채 시즌처럼 특정 시기에 사용자가 늘어날 수 있는 서비스 특성을 고려해, 서버 세션에 의존하지 않는 무상태 인증 방식을 적용했습니다.  
이를 통해 서버 자원 사용을 줄이고, 확장성 측면에서도 유리한 구조를 선택했습니다. 

### MySQL & Redis
회원 정보, 자기소개서 등 핵심 데이터는 트랜잭션과 무결성이 중요한 관계형 데이터로 관리하기 위해 MySQL을 사용했습니다.  
반면 리프레시 토큰처럼 TTL이 필요하고, 빠른 접근 속도가 중요한 데이터는 Redis에 저장해 성능을 높였습니다. 

## 담당 역할
- REST API 설계
- DB 스키마 구축
- OAuth2, JWT, Spring Security를 활용한 소셜 로그인 구현
- JPA 기반 CRUD 기능 구현
- 자소서 및 스펙 관리 기능 개발 

## 문제 해결 경험 1
### 주제
DTO 도입을 통한 데이터 전송 최적화

### 문제 상황
엔티티 전체를 직접 노출할 경우, 보안상 불필요한 필드까지 외부로 전달될 수 있고, 화면에서 필요하지 않은 데이터까지 함께 내려가 비효율이 발생할 수 있었습니다. 

### 해결 방식
화면 목적과 API 응답 목적에 맞는 전용 DTO를 설계해 필요한 데이터만 전달하도록 구조를 개선했습니다. 

### 의미
이 경험은 단순 구현에서 나아가, 계층 분리와 API 응답 설계의 중요성을 이해하고 적용한 사례입니다.

## 문제 해결 경험 2
### 주제
Redis를 활용한 토큰 관리 최적화

### 문제 상황
리프레시 토큰을 RDBMS에서 관리하면 잦은 I/O가 발생하고, 토큰 재발급 요청 시 응답 속도 저하 우려가 있었습니다. 

### 해결 방식
Redis를 도입해 리프레시 토큰을 저장하고, TTL 기능을 활용해 자동 만료가 되도록 구성했습니다.  
이를 통해 토큰 관리 성능과 운영 효율성을 높였습니다. 

### 의미
저장소의 특성에 따라 적절한 기술을 선택하는 경험이었고, 성능과 관리 편의성을 함께 고려한 사례입니다.

## 문제 해결 경험 3
### 주제
공통 필터를 통한 인증 누락 해결

### 문제 상황
초기에는 로그인 시점에만 인증이 처리되어 이후 요청에서는 사용자 권한이 정상적으로 인식되지 않는 문제가 발생할 수 있었습니다. 

### 해결 방식
모든 요청을 가로채는 공통 인증 필터와 예외 핸들러를 구현하여, 각 요청마다 인증 상태를 확인하고 일관되게 권한을 처리할 수 있도록 개선했습니다. 

### 의미
보안 기능을 단순 로그인 구현에서 끝내지 않고, 요청 전반의 흐름 속에서 설계한 경험입니다.

## 4. Personality and Working Style

### 사용자 가치 중심
이 지원자는 기술 자체보다, 그 기술이 사용자에게 어떤 가치를 주는지를 중요하게 생각합니다.  
예를 들어 FLOFI에서는 단순 음악 변환이 아니라 몰입 경험까지 고려해 타이머와 백색소음을 추가했고, Workfolio에서는 취업 준비 과정에서 흩어진 정보를 한곳에서 관리할 수 있도록 통합 경험에 집중했습니다.  

### 구조적으로 문제를 해결하는 태도
문제가 생겼을 때 임시방편으로 처리하기보다, 원인을 분석하고 구조를 바꾸는 방식으로 접근합니다.  
예를 들어 데이터 포맷을 MIDI로 바꾸거나, 토큰 저장소를 Redis로 분리하거나, 트랜잭션을 도입해 무결성을 확보한 경험이 있습니다.   

### 협업과 조율
FLOFI 프로젝트에서는 리더 역할을 맡아 Github, Notion, 간트차트를 활용해 협업 과정을 이끌었습니다.  
기술 구현뿐 아니라 팀 단위의 진행 관리 경험도 보유하고 있습니다.

## 5. Q&A Guidance for Chatbot

### 답변 원칙
- 반드시 실제 경험에 기반해 답변할 것
- 없는 경험은 추측하거나 생성하지 말 것
- 기술 이름만 나열하지 말고, 문제-해결-결과 흐름으로 설명할 것
- 질문이 모호하면 가장 관련 높은 프로젝트를 기준으로 답변할 것

### 추천 답변 구조
1. 어떤 프로젝트에서의 경험인지 설명
2. 어떤 문제 또는 목표가 있었는지 설명
3. 어떤 기술과 방식으로 해결했는지 설명
4. 그 경험을 통해 무엇을 배웠는지 정리

### 질문 유형별 연결 가이드
- 협업 / 리더십 질문 → FLOFI 중심
- DB / 트랜잭션 / 무결성 질문 → Movie Web 중심
- Spring / 인증 / JWT / Redis 질문 → Workfolio 중심
- 사용자 중심 개선 경험 → FLOFI, Workfolio 중심
- 기술 선택 이유 질문 → 각 프로젝트의 Tech Why 항목 중심
